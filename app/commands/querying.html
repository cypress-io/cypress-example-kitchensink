Best Practices: Locating elements

Prefer dedicated data-test attributes to CSS class names and element IDs. See detailed discussion at Best Practices: Selecting elements

The following examples are based on the submit button seen in the right side 


  # Worst - too generic, no context
  driver.find_element_by_css_selector("button")

  # Bad. Coupled to styling. Highly subject to change.
  driver.find_element_by_css_selector(".btn.btn-large")

  # Average. Coupled to the `name` attribute which has HTML semantics.
  driver.find_element_by_css_selector("[name=submission]")

  # Better. But still coupled to styling or JS event listeners.
  driver.find_element_by_css_selector("#main")

  # Slightly better. Uses an ID but also ensures the element
  # has an ARIA role attribute
  driver.find_element_by_css_selector("#main[role=button]")

  # Much better. But still coupled to text content that may change.
  driver.find_element_by_css_selector("Submit")

  # Best. Insulated from all changes.
  driver.find_element_by_css_selector("[data-test=submit]")
